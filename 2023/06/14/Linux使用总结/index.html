安装GPU版本的PyTorch

建议先进入torch的wheel下载网站查看版本对应的情况。

拿我自己举例，服务器下安装的CUDA的版本是12.1，但是PyTorch的官网上显示目前支持的CUDA版本为11.7和11.8，秉持着用新不用旧的原则，就运行了官网提供的CUDA 11.8的Command，但是没有安装成功，然后找了半天发现自己的python版本是3.7，在上面wheel网站里发现3.7版本的python最高支持的CUDA版本为11.7，于是在官网中的Compute Platform这一项选择了CUDA 11.7，然后在运行官网提供的新Command，待安装完成之后执行torch.cuda.is_available()，返回True。

多个项目使用同一个数据集

在训练深度学习模型时，很多模型需要用到同一个数据集，而很多模型都要求在当前算法文件夹中存放数据集，为了减少内存浪费，可以在当前文件夹下使用软连接来连接到数据集源文件，就可以不需要复制多个数据集了。

    Pycharm远程连接服务器

  • 确保自己的Pycharm版本为professional版本
  • 创建远程连接：Tools—>Deployment—>Configuration—>SFTP协议—>配置服务器的IP、端口、用户名、密码等相关信息
  • 进行本地项目与远程项目之间的文件夹路径映射：在刚才的SFTP协议中选择Mappings添加映射，例如我们将本地的D:\workspace\PycharmProjects\Code路径映射到远端的/home/jeeby/Code路径
  • 设置文件自动上传（可选）：Tools—>Deployment—>Automatic Upload，默认是有修改就自动上传，可以在Tools—>Deployment—>Options中设置按了ctrl+s上传
  • 验证是否连接成功：Tools—>Deployment—>Browse Remote Host，如果能打开服务器文件目录说明连接成功
  • 配置Python解释器
    • 查询服务器中所使用的的Python解释器路径：which python命令，注意虚拟环境的话需要先进入虚拟环境再使用前述命令
    • 本地Pycharm中配置Python解释器：File—>Settings—>Project—>Project Interpreter—>右上角设置图标Add—>SSH Interpreter—>Existing server configuration—>Move—>配置Interpreter和Sync folders（和前面配置的文件夹路径映射一致即可）
  • 本地启动服务器终端：Tools—>Start SSH session
