克隆项目

  1. 新建一个文件夹
  2. 打开CMD或者右键Git Bash Here
  3. 命令行中输入git init，此时刚创建的文件夹中出现一个.git文件夹
  4. 先在GitHub中复制URL，然后命令行窗口中输入git clone [URL] [需要新建的文件夹名]，回车，等待下载完成即可

Fork项目

  1. 点击想要fork的原仓库的fork按钮，执行完这个操作后会将原仓库的文件、提交历史、issues和其余内容复制到自己的github仓库中；
  2. 采用git clone命令在本地工作区克隆原仓库的代码；
  3. 采用git remote add <remote_name> <remote_url>向当前仓库中添加自己的远程仓库；
  4. 后续更改任何东西都可以在本地完成，如add、commit一系列的操作，然后通过push命令推到自己的远程仓库；
  5. 如果希望对方接受你的修改，可以通过发送pull requests给对方。如果对方接受，则会将你的修改内容更新到仓库中。

子模块详解

git submodule命令用于管理包含其他Git仓库的项目。git submodule命令对于大型项目或需要将外部库集成到项目中的情况非常有用。 通过使用子模块，你可以将外部库作为你的项目的一部分来管理，而不必将其直接合并到主仓库中。

初始化子模块，git submodule init命令会初始化配置文件中的所有子模块。它会根据.gitmodules文件中的信息设置子模块的URL和路径，但不会下载子模块的内容。

git submodule update命令会从子模块的远程仓库中拉取子模块的内容，并将其更新到.gitmodules文件中指定的提交。

git submodule add <repo-url> [<path>]命令会将指定的Git仓库作为子模块添加到当前仓库中。<repo-url> 是子模块的仓库地址，<path> 是子模块在主仓库中的路径（可选，如果不指定，默认使用子模块仓库的名称作为路径）。

git submodule deinit <path>命令将子模块从.git/config文件中移除，并删除子模块目录中的文件。git rm <path>命令将子模块的引用从主仓库中删除，并提交更改。

git submodule命令列出当前仓库中的所有子模块，以及它们的提交哈希和路径。

git submodule update --recursive --remote命令更新所有子模块。

git submodule status命令显示子模块的当前状态，包括当前的提交哈希和路径，是否有未提交的更改。
